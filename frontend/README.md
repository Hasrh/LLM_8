# Security Audit Frontend

This frontend is a lightweight dashboard for the security benchmark outputs generated by this project.

## Features

- Upload any benchmark `results.json` file from your local machine.
- Filter by suite, search target names, enforce minimum score, and show only passing runs.
- View summary metrics: run count, average score, pass rate, unique targets.
- Inspect score distribution and suite-wise comparison.
- Browse top runs in a table sorted by score.
- Analyze a new repo and compare findings across:
  - GPT Direct (`mode=direct`)
  - OpenCode Baseline (`mode=baseline`)
  - OpenCode + RAG (`mode=rag`)

## Run locally

```bash
npm install
npm run dev:full
```

Then open the URL shown by Vite (usually `http://localhost:5173`).

`dev:full` starts:
- Vite frontend
- local analysis API (`server.mjs`) on `http://127.0.0.1:8787`

## Build for production

```bash
npm run build
npm run preview
```

## Input format

The dashboard expects an array of objects like:

```json
[
  {
    "suite": "rag",
    "target": "/path/to/repo",
    "ok": true,
    "run_dir": "/path/to/run",
    "score": 0.8,
    "passed": false,
    "provider_id": "opencode",
    "model_id": "qwen3.6-plus-free"
  }
]
```

A sample file is included at `public/sample-results.json`.

## Comparative analysis API

The UI calls `POST /api/analyze-compare` and the local API runs:

- `opencode analyze --mode direct` (with GPT model)
- `opencode analyze --mode baseline`
- `opencode analyze --mode rag`

### Requirements

- `opencode` command available in PATH (or set `OPENCODE_BIN`)
- For GPT direct mode, provider/model access configured in your OpenCode setup
- For RAG mode, a valid controls corpus JSON path
- If using vector DB in RAG, matching backend configured (Qdrant/Pinecone)
